# Movies API Service

## Общее описание
**Цель сервиса:** Предоставление каталога фильмов, жанров и персон с разграничением доступа к новому контенту на основе подписки пользователя.

**Стек технологий:** 
- **Язык:** Python 3.14+
- **Фреймворк:** FastAPI
- **База данных/Поиск:** Elasticsearch (основное хранилище и поиск)
- **Кэширование:** Redis
- **Сервер приложений:** Gunicorn

**Зависимости:**
- Внешние API: Не использует.
- Инфраструктурные зависимости: Elasticsearch, Redis.

## Бизнес-логика и правила (Business Logic)

### Ключевые сущности
- **Movie (Фильм):** Основная единица контента с метаданными (дата создания, название и т.д.).
- **Genre (Жанр):** Категория фильма для фильтрации и группировки.
- **Person (Персона):** Актеры, режиссеры и другие участники съемочного процесса.

### Основные бизнес-правила
- **Доступ к контенту:** 
    - Если фильм «новый» (дата создания позже чем `non_subscriber_content_age` лет от текущей даты), то доступ к нему имеют только пользователи с ролью `subscriber`.
    - Пользователи без подписки могут просматривать только старый контент.

### Жизненный циклы
Сервис работает преимущественно в режиме чтения данных из Elasticsearch через слой кэширования Redis:
`Запрос API` $\rightarrow$ `Проверка Кэша (Redis)` $\rightarrow$ `(если нет) Запрос к Индексу (Elasticsearch)` $\rightarrow$ `Валидация прав доступа` $\rightarrow$ `Ответ`.

### Интеграции
- **Входящие:** Сервис вызывается внешними клиентами через REST API.
- **Исходящие:** 
    - **Elasticsearch:** Запросы на получение данных по ID и полнотекстовый поиск (по событию API-запроса).
    - **Redis:** Чтение/запись закэшированных ответов для снижения нагрузки на Elasticsearch (по событию API-запроса).
